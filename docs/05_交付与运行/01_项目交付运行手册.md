# LiaoClaw 项目交付运行手册（Windows + macOS）

适用场景：把项目完整发给别人，对方拿到后可直接运行。

## 1. Windows 一分钟启动（最短可用）

在项目根目录执行：

```powershell
py -3.10 -m venv .venv
.\.venv\Scripts\Activate.ps1
pip install -e ".[dev]"
Copy-Item .env.ps1.example .env.ps1
```

编辑 `.env.ps1`，至少填一个 Key：

1. `$env:ANTHROPIC_API_KEY = "你的key"`
2. 或 `$env:OPENAI_API_KEY = "你的key"`

启动 Web（简短版）：

```powershell
liaoclaw-web --provider anthropic --model-id glm-4.5-air
```

浏览器打开：`http://127.0.0.1:8787`

## 2. macOS 一分钟启动（最短可用）

在项目根目录执行：

```bash
python3 -m venv .venv
source .venv/bin/activate
pip install -e ".[dev]"
cp .env.example .env
```

编辑 `.env`，至少填一个 Key：

1. `ANTHROPIC_API_KEY=你的key`
2. 或 `OPENAI_API_KEY=你的key`

启动 Web（简短版）：

```bash
liaoclaw-web --provider anthropic --model-id glm-4.5-air
```

浏览器打开：`http://127.0.0.1:8787`

## 3. 简短命令（推荐）

Windows：

1. Web：`liaoclaw-web --provider anthropic --model-id glm-4.5-air`
2. CLI：`. .\.env.ps1; liaoclaw --mode interactive --provider anthropic --model-id glm-4.5-air`
3. IM 长连接：`.\dev.ps1 -Mode im -Transport longconn`

macOS：

1. Web：`liaoclaw-web --provider anthropic --model-id glm-4.5-air`
2. CLI：`bash ./dev.sh --mode cli`
3. IM 长连接：`bash ./dev.sh --mode im --transport longconn`

## 4. Skills 怎么调用

推荐标准目录结构：

```text
skills/
  quick_check/
    SKILL.md
```

兼容写法（仍支持）：

1. `.liaoclaw/skills/*.md`
2. `.Liaoclaw/skills/*.md`（兼容旧目录名）

每个 skill 文件支持 frontmatter：`name`、`command`、`description`。

OpenClaw 常用可选字段（已兼容）：

1. `metadata.openclaw.os`
2. `metadata.openclaw.requires.bins`
3. `metadata.openclaw.requires.anyBins`
4. `metadata.openclaw.requires.env`
5. `metadata.openclaw.requires.config`
6. `user-invocable`
7. `disable-model-invocation`
8. `command-dispatch: tool` + `command-tool` + `command-arg-mode`

调用指令（CLI / IM / Web）：

1. 先用 `/help` 查看可用 skills 命令
2. 用 `/<command>` 触发，例如：
   `/quick-check`

当前行为说明（很重要）：

1. `/<command>` 会触发 skill 命令并返回技能模板文本
2. 它不会直接自动发起一轮模型推理
3. 真正要做任务时，继续发送你的业务请求即可

Web 端说明：

1. Web 输入框已支持斜杠命令分发（如 `/help`、`/quick-check`）
2. 与 CLI/IM 一样，技能命令先返回技能模板文本，再继续发送任务内容

## 4.1 Skills 手动激活

当你新增或修改 `SKILL.md` 后：

1. CLI：需要重启 `liaoclaw` 进程（`/new` 只清上下文，不重新扫描 skills）。
2. Web：可点击“新会话”触发重建会话，或直接重启 `liaoclaw-web`。
3. IM：建议重启 `liaoclaw-im` 或 `dev.ps1 -Mode im` 进程，确保重新加载 skills。

## 4.2 Skills 功能测试（可直接复制）

1. 仓库已自带可测试技能：

```powershell
Get-Content .\skills\quick_check\SKILL.md
```

2. 启动 CLI：

```powershell
. .\.env.ps1
liaoclaw --mode interactive --provider anthropic --model-id glm-4.5-air
```

3. 在 CLI 中验证：

```text
/help
/quick-check
```

说明：上述两条只验证 skills 分发链路，不会触发模型请求。

预期：

1. `/help` 里出现 `[skill] /quick-check`
2. `/quick-check` 返回“已应用技能 ...”文本

4. 启动 Web 验证：

```powershell
liaoclaw-web --provider anthropic --model-id glm-4.5-air
```

在网页输入框依次发送：

```text
/help
/quick-check
```

说明：上述两条同样不会触发模型请求。

预期同 CLI：都能返回技能命令结果。

5. 自动化测试（开发者）：

```powershell
pytest tests/test_coding_agent_resources.py tests/test_coding_agent_web.py -q
```

如果显示通过，说明 skills 的加载、过滤和 Web 分发都正常。

## 5. 与 OpenClaw Skills 的区别（是否“标准”）

结论：当前实现已兼容 OpenClaw 核心技能规范，但不是完整的 OpenClaw 网关运行时。

1. 已兼容：`<skill>/SKILL.md`、YAML frontmatter、`metadata.openclaw` 常用 gating 字段
2. 已兼容：`user-invocable`、`disable-model-invocation`、`command-dispatch: tool`
3. 未覆盖：OpenClaw 专有的网关管理能力（例如 ClawHub 安装命令、skills watcher、完整 agents allowlist 语义）
4. 对当前项目而言，这套实现已足够按 OpenClaw 技能格式编写与运行

如果你后续要对齐 OpenClaw 风格，可以再做一层 skill schema 兼容转换器。

## 6. 什么时候用长命令

仅在默认值不满足时加参数，例如端口冲突：

```bash
liaoclaw-web --provider anthropic --model-id glm-4.5-air --host 127.0.0.1 --port 8788
```

说明：`liaoclaw-web` 默认是 `127.0.0.1:8787`。

## 7. 可选：飞书 IM 显式命令

先安装：

```bash
pip install -e ".[dev,feishu]"
```

Windows 显式命令：

```powershell
liaoclaw-im --platform feishu --transport longconn --provider anthropic --model-id glm-4.5-air --feishu-app-id $env:FEISHU_APP_ID --feishu-app-secret $env:FEISHU_APP_SECRET
```

macOS 显式命令：

```bash
liaoclaw-im --platform feishu --transport longconn --provider anthropic --model-id glm-4.5-air --feishu-app-id "$FEISHU_APP_ID" --feishu-app-secret "$FEISHU_APP_SECRET"
```

## 8. 常见问题

1. `Missing ANTHROPIC_API_KEY`：没配 Key，补 `.env` 或 `.env.ps1`。
2. `Unknown model`：模型名写错，先用 `anthropic + glm-4.5-air`。
3. `WinError 10053`：浏览器断开连接常见日志，一般不影响使用。
4. `Address already in use`：端口占用，换 `--port 8788`。
5. `liaoclaw 不是内部或外部命令`：没激活 `.venv` 或没安装依赖。

## 9. 交付前自检清单

1. `python --version` 显示 `3.10+`
2. `pip install -e ".[dev]"` 成功
3. `.env` 或 `.env.ps1` 已填真实 Key
4. `liaoclaw-web --provider anthropic --model-id glm-4.5-air` 可启动
5. `CLI /help` 可看到 `[skill]` 命令
6. 页面或 CLI 能正常对话
